Some Key Takeaways

Tags: Some tags require closing, and some tags do not, which are considered VOIDS.
Elements: This is what an HTML is essentially made up of.
Attributes: Used with specific tags to make things act, appear, or overall just process specifically different depending on the instruction.
HTML: Hyper Tet Mark-up Language
Inline: An element that is in-line with the contents.
Block: An element that create an invisible box that can be adjusted.

The Tree

The next thing is to understand how the structure of the webpage cascades. The tree-like description can be misleading, and does not necessarily speak to the actual visual representation, but more of the way the information branches into each subsection. This basically means that each section may contain limbs which then branch of into smaller sections which delve deeper into each subsection.

CSS

CSS stands for Cascading Style Sheets. CSS is a styling language, which means that we can write code to change the way a webpage looks and feels. The cascading portion of the name suggests that it can either transfer into the section, or classes can be used in order to specify the element to be styled. CSS does not necessarily need to be memorized, but we do need to know where to go when we need help.
